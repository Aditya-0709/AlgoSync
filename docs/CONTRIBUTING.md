# Contributing

Before creating a PR

✔ Read SOFTWARE_DESIGN_DOCUMENT.md

Rules

- Never duplicate code.

- Never mix UI and business logic.

- Never hardcode markdown.

- Never rewrite working modules.

- Keep functions small.

- Use reusable utilities.

- Every feature should have one responsibility.

- Build succeeds

- LeetCode works

- GFG works

- GitHub Sync works

- README generation works
